# 리액트 네이티브 도서 집필 계획 및 진행 현황

이 문서는 책 전체의 집필 계획, 장별 진행 상태, 작업 메모를 한곳에서 관리하기 위한 루트 문서입니다.

## 작업 원칙

- 집필은 장(Chapter) 단위로 점진적으로 진행합니다.
- 각 장은 별도 폴더에서 관리합니다.
- 각 장 폴더에는 워드 원고 파일과 예제 소스코드를 함께 둡니다.
- 예제 코드는 장의 설명 흐름과 동일한 순서로 추가합니다.
- 집필 상태가 바뀌면 이 문서를 먼저 갱신합니다.

## 폴더 구조

```text
.
├─ BOOK_PROGRESS.md
├─ IT_책_집필_워드_템플릿_ko.docx
├─ make_it_book_template.py
└─ chapters/
   ├─ 01_장제목/
   │  ├─ manuscript/
   │  │  └─ 01_장제목.docx
   │  ├─ examples/
   │  └─ notes.md
   └─ 02_장제목/
      ├─ manuscript/
      ├─ examples/
      └─ notes.md
```

## 장별 작업 체크리스트

각 장은 아래 순서로 진행합니다.

1. 장 제목과 학습 목표 확정
2. 세부 목차 작성
3. 예제 코드 초안 작성
4. 워드 원고 작성
5. 코드와 원고 교차 검토
6. 장 마무리 정리 및 상태 업데이트

## 메모

- 워드 원고 파일은 각 장의 `manuscript/` 폴더에서 수동으로 작성합니다.
- 예제 실행 방법은 장 폴더의 `notes.md`에 함께 정리합니다.
- 장 제목이 확정되면 폴더 이름을 `01_리액트_네이티브_소개`처럼 실제 제목 기반으로 변경합니다.
