# 7장 메모: 목록 정렬하기
## 장 목표

- 정렬 기준 상태에 따라 `FlatList` 순서를 바꾸는 방법을 익힌다.
- 원본 데이터는 그대로 두고, 화면에 보여 줄 정렬 결과만 계산하는 구조를 이해한다.
- `이름순`, `팔로잉 우선` 같은 정렬 옵션을 버튼으로 전환해 본다.
- chapter 06의 검색 가능한 목록을 정렬 가능한 목록으로 확장한다.

## 구성 초안

1. 왜 목록 정렬이 필요한가
2. 정렬 기준 상태 만들기
3. 필터 결과를 정렬된 배열로 다시 계산하기
4. 정렬 버튼 UI 연결하기
5. 실행 결과 확인
6. 정리

## 실습 메모

- chapter 07은 chapter 06의 검색 구조를 유지한 채 정렬 기능을 추가한다.
- 이번 장의 핵심은 `검색 결과 -> 정렬 결과 -> FlatList` 흐름을 이해하는 것이다.
- 원본 배열을 직접 재배치하기보다 화면용 결과를 따로 계산하는 방식에 익숙해지는 것이 중요하다.
